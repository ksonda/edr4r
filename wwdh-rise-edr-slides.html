Examples verified against the live service, June 10, 2026

Three names, one idea

  • RISE — Reclamation's public data catalog (data.usbr.gov): storage, elevations, releases, canal flows, weather… by location and parameter
  • EDR — OGC API "Environmental Data Retrieval": an open standard for asking data questions with plain URLs
  • WWDH — the Western Water Datahub republishes RISE through that standard, as the collection rise-edr

Why bother? Learn one URL grammar and it also works for SNOTEL snowpack, USACE reservoirs, and PRISM climate on the same server.

Anatomy of a request


https://api.wwdh.internetofwater.app / collections/rise-edr/locations/3514 ? f=json & parameter-name=3
└──────────────── base ─────────────┘ └───────────── path ──────────────┘   └───────── query ────────┘
  • base — always https://api.wwdh.internetofwater.app
  • path — what kind of thing you want
  • query — after ?, joined by &: format, dates, parameters, bounding box

The one switch to remember: f=

f=What you getUse it for
htmlReadable page, often a map or table Browsing, orienting, sharing a link
jsonGeoJSON / CoverageJSON Downloads, scripts, GIS, Power Query
csvSpreadsheet table (feature lists) Straight into Excel

HTML pages are a window onto the data. Same URL, different f= — browse first, automate later.

Using curl? Just remember single quotes:
curl -sS 'https://api.wwdh.internetofwater.app/collections/rise-edr?f=json'

Orientation: the collection page

https://api.wwdh.internetofwater.app/collections/rise-edr

What can you ask RISE? data_queries

QueryPathQuestion it answers
locations/locations Where are the monitoring locations? Data for one of them?
items/items Searchable, filterable catalog of RISE locations
cube/cube Data for everything inside a bounding box
area/area Data for everything inside a polygon

A route not advertised in data_queries will error — e.g. RISE has no /position. Check the collection document first.

Find locations: the map

/collections/rise-edr/locations   (constrain with &bbox=min_lon,min_lat,max_lon,max_lat)

Lon/lat order; western-US longitudes are negative. Switch to f=json for GeoJSON with location ids.

Find locations: the items catalog

/collections/rise-edr/items?f=html&bbox=-115,33,-111,35&locationTypeName=Lake%2FReservoir&limit=20

7 lake/reservoir locations in a Lower Colorado bbox, incl. Lake Havasu (3515).

Items: the three filter tools

  • bbox= — spatial window
  • locationTypeName=Lake%2FReservoir — any property from /queryables works as a filter (%2F is an encoded /)
  • limit= — row cap. Start generous: some filters apply after the limit

Spreadsheet download: change f=html → f=csv — same filtered table, opens in Excel, with capacities and datum notes where RISE has them.

One location's detail page: /items/3514 → Lake Mead Hoover Dam and Powerplant.

Get data: one location, one URL

Lake Mead (3514), daily storage (3), January 2023:

https://api.wwdh.internetofwater.app/collections/rise-edr/locations/3514?f=json&datetime=2023-01-01/2023-01-31&parameter-name=3
  • datetime= — instant or range (2023-01-01/2023-01-31); omit it and you may get the whole period of record. Start narrow.
  • parameter-name= — one or more ids, comma-separated: 3,1834
  • curl: curl -sS -o mead.json '<URL>'

Reading CoverageJSON


{ "type": "Coverage",
  "domain": {
    "domainType": "PointSeries",
    "axes": {
      "x": {"values": [-114.7]},
      "y": {"values": [36.0]},
      "t": {"values": ["2023-01-01...",
                       "2023-01-02..."]}
    }
  },
  "ranges": {
    "3": { "axisNames": ["t"],
           "values": [8730345,
                      8732112] } } }

Pair them up positionally:
1st time ↔ 1st value, 2nd ↔ 2nd…

tparam 3 (af)
2023-01-018,730,345
2023-01-028,732,112

CSV is advertised on data queries but tested empty — treat f=json as the dependable format for time series.

Many locations at once: /cube

All RISE storage series in a box around Lake Havasu, calendar 2024:

/collections/rise-edr/cube?f=json&bbox=-114.3,34.1,-113.5,34.4&datetime=2024-01-01/2024-12-31&parameter-name=3
  • Returns a CoverageCollection: one coverage per location
  • Each cube fans out to the upstream RISE API — keep bbox tight and dates short, then widen one thing at a time
  • Broad cube failing? Fall back: ids from /items, then /locations/{id} one at a time

Polygons: /area

WKT polygon in coords=; every space becomes %20:


POLYGON((-115 35,-113 35,-113 37,-115 37,-115 35))
/collections/rise-edr/area?f=json&coords=POLYGON((-115%2035,-113%2035,-113%2037,-115%2037,-115%2035))&datetime=2024-01-01/2024-01-31&parameter-name=3
  • Ring must close (repeat the first coordinate)
  • Very long URLs hit limits — simplify the boundary, or if a box is close enough, prefer cube

Cross-agency crosswalk: parameterGroups

/collections?f=json maps shared concepts to each collection's ids:


{ "name": "Lake/Reservoir Storage",
  "members": {
    "rise-edr":  ["1470", "56", "51", "47", "3"],
    "usace-edr": ["Flood Storage", "Conservation Storage", ...] } }

2024 storage near Parker Dam = two parallel cubes, same bbox:
rise-edr → parameter-name=3  |  usace-edr → parameter-name=Flood%20Storage

Expect timestep differences (RISE daily, USACE irregular) — aligning them is an analysis step, not an API option.

The whole workflow


1. ORIENT      /collections/rise-edr                       (browser)
2. PARAMETERS  /collections/rise-edr?f=json                -> pick exact ids
3. LOCATIONS   /items?f=html&bbox=...&{queryable}=...      -> note ids; f=csv to download
4. ONE SERIES  /locations/{id}?f=json&datetime=...&parameter-name=...
5. MANY SERIES /cube?f=json&bbox=...&datetime=...&parameter-name=...
6. CROSSWALK   /collections?f=json -> parameterGroups -> repeat on sibling

Start small — one location, one parameter, one month.
Widen one dimension at a time.

When it doesn't work

SymptomFix
404 on a query routeRoute not advertised — re-check data_queries
400 on /areaEncode spaces %20; close the ring
204 / empty bodyValid but no match — different parameter, location, or dates
Empty CSV from data queryUse f=json; CSV works on /items
500 / slow /cubeShrink bbox, shorten dates, one parameter
Empty featuresRaise limit, drop filters one at a time
Shell error before requestSingle-quote the URL
